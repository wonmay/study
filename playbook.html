第一步：识别目标客户
核心目标：用 3 分钟内判断对方是否是值得深入跟进的目标客户
Day 1
关键动作
电话前10秒确认对方身份：是否为CTO/技术总监/AI负责人
用一句话切入：以"AI服务优化顾问"身份而非推销员
问关键问题："贵公司现在有在调用大模型 API 吗？"
快速判断月消耗级别：≥10万 Tokens = 继续，否则礼貌结束
约定下次沟通时间，获取微信（加好友比电话更高效）
常见错误
一上来就说"我们比官网便宜"→ 立即被挂断
没确认是否在用 Tokens 就狂讲方案 → 浪费双方时间
找到前台/HR → 没有决策权，直接放弃这条线
10分钟内无法确认月消耗级别 → 本次价值存疑
介绍产品超过3分钟 → 客户失去耐心
必备话术 · 电话切入开场白
"您好，我是 rhcloud AI 服务团队的 XX，我们专注帮做 AI 业务的中小企业优化大模型调用成本和稳定性。 我看到贵公司在 AI 方向有一些布局，所以冒昧联系一下——请问您这边现在有在调用大模型 API 吗？大概月均是什么量级？"
✅ 成功信号（出现以下信号立即进入下一步）
确认在用 Tokens 月消耗 ≥ 10万 决策人本人接听 主动问"你们平台怎么样" 同意约下次见面
第二步：直击业务痛点
核心目标：让客户自己说出痛苦——稳定性、成本或速度，至少命中其一
Day 1~3
核心销售原则
SPIN 法则：先问现状（Situation）→ 问问题（Problem）→ 放大影响（Implication）→ 引导需求（Need-payoff）。 不是你去"讲"痛点，而是引导客户自己"说"出来，他说出的比你说的更有力量。
三大痛点挖掘路径
稳定性路径：问"高峰期推理延迟有没有影响到用户体验？"
成本路径：问"现在大模型 API 的费用大概占整体运营成本多少比例？"
速度路径：问"用户输入到看到第一个字的响应时间大概多少秒？"
多模型路径：问"现在只用一家的模型吗？有没有想过切换时的麻烦？"
绝对不能做的事
不要替客户描述痛点（"你们肯定延迟很高吧"→ 引发抵触）
痛点挖掘阶段不讲方案（先挖坑，再填坑）
痛点没挖深就报价 → 客户没有"必须买"的理由
一次问超过3个问题 → 像审讯，客户关闭心防
必备话术 · 痛点深挖三连问
"我先了解一下您们现在的使用情况——高峰期的时候，大模型响应速度大概是多少秒？ 有没有出现过超时或报错的情况？这对您们的业务有影响吗？ 如果这个问题能解决掉，您觉得对业务帮助有多大？"
✅ 成功信号
客户主动抱怨过稳定性 客户说"成本确实有点高" 客户问"你们能解决这个问题吗" 客户说"目前没什么问题"→慎重，需要继续挖
第三步：给出平台方案
核心目标：精准对应客户痛点给出方案，用 ROI 数字而非功能列表说话
Day 3~7
黄金法则
方案不是"我们有什么"，而是"你的问题 → 我们怎么解决 → 解决后你能得到什么（数字）"。 永远用 ROI 收益而不是技术参数说话。
痛点 → 方案对应关系
稳定性差 → 专有集群方案，稳定性 99.9%+
成本偏高 → 智能路由自动选最优性价比，综合降本 30%
响应慢 → 高 TPM 专有通道，速度提升 60%
多模型切换麻烦 → 统一接口，10+ 模型一键切换
迁移风险 → 只改一行 base_url，无需重构系统
现场 ROI 快算法
问客户月均 Token 消耗量
问现有单价（元/百万 Tokens）
当场计算：节省额 = 月用量 × 单价差 × 12
加上稳定性收益（宕机损失估算）
一句话总结："您每年能省 XX 万，同时稳定性提升"
必备话术 · 方案呈现+ROI引导
"根据您刚才说的情况，我给您梳理一下。您现在的问题主要是两个：一是高峰期响应不稳，二是成本有点高。 我们的方案是这样的：第一，给您接入专有集群，不跟其他用户共享，高峰期照常稳定； 第二，我们的智能路由会自动帮您选最划算的模型，综合下来成本能降 30% 左右。 按您每月 XX 万 Tokens 的量算，一年能省 XX 万。最重要的是，迁移只要改一行代码， 您的技术团队当天就能搞定，零风险。您觉得先做个小测试怎么样？"
✅ 成功信号
客户问"怎么接入" 客户问"测试账号怎么申请" 主动问"合同怎么签" 让技术同事一起加入沟通
第四步：推动迁移转化
核心目标：以小流量 POC 测试降低客户决策风险，用数据说话驱动正式签约
Day 7~15
核心策略：小步快跑，数据驱动
不要让客户一次性把所有流量都迁过来。建议"先迁 1% → 跑 3 天 → 看数据 → 再扩量"。 数据会替你说话，客户自己看到效果就会主动要求全量迁移。
POC 推进三步法
提供 POC 测试账号，免费体验 7 天专有集群
帮客户技术团队接入（只改 base_url 一行代码）
先迁移 1% 流量，观察延迟、稳定性、成本数据
3天后发送数据对比报告，客户自己看结果
锁定价格：限时额度或阶梯价，推动签约
逼单技巧（正确使用）
限时额度：本月额外赠送 XX 万 Tokens 测试包
阶梯价锁定：本季度签约享受优先价格
稀缺性："专有集群节点资源有限，建议优先锁定"
不要说"快要结束了"之类的假话 → 伤信任
不要每天催 → 让客户觉得产品不好卖
必备话术 · POC 邀请 + 锁定
"我给您申请一个 7 天的专有集群测试账号，您的技术同学只需要改一行代码就能接入，完全不影响现有业务。 先把 1% 的流量切过来跑 3 天，我帮您对比一下延迟、稳定性和成本数据， 您看完数据再决定要不要全量迁移。 本月我们还有额外的测试 Tokens 赠送，您看什么时候方便让技术团队对接一下？"
✅ 成功信号
同意 POC 测试 拉入技术负责人群 测试数据超预期 主动问报价和合同 同意阶梯价格方案
销售不是说服，是帮助客户发现他自己的问题，并给出他本就需要的解决方案
每一次被拒绝都是因为你还没找到客户的真实痛点 · 坚持挖，成单只是时间问题

跟单节奏管理

每个时间节点的精准动作、目标与配套话术，15 天从初接触到签约

节奏黄金原则
太快 = 给客户压力引发抵触；太慢 = 客户遗忘、竞品趁虚而入。每次跟进必须带新价值（数据/案例/优惠），不做无意义"催单"。
Day 1 · 初次触达 电话切入 · 建立联系 必须完成
目标：判断是否目标客户，建立初步信任，获取微信/企业微信
打电话，用开场话术确认是否在用 Tokens，对方身份是否 CTO/技术负责人
通话结束后立即发微信：发送公司简介一张图 + 核心价值3行字
CRM 记录：月用量级别、决策人姓名、主要痛点关键词
微信跟进话术（通话后1小时内发送）
"您好，我是刚才通话的 XX。我们 rhcloud 是阿里云核心代理商，专注帮 AI 业务团队优化大模型成本和稳定性。 根据您的情况，我整理了一份「大模型成本优化方案」，待您有空看一眼。如有问题随时沟通 😊"
Day 3 · 二次跟进 发送 ROI 对比 + 行业案例 价值输出
目标：让客户产生"这对我有价值"的认知，引发深入交流兴趣
发送该客户所在行业的 ROI 对比数据（从行业作战室获取）
发送同行业已有客户的脱敏案例（月省 XX 万、延迟降低 XX%）
结尾轻量提问，引导回复（避免石沉大海）
Day 3 跟进话术
"王总好！我整理了一份同行业客户的对比数据，供您参考：某 AI 视频公司接入我们平台后， 高峰期延迟从平均 2.8 秒降到 0.9 秒，月 Token 成本降了 31%。 我估算了一下，按您们的月用量，一年大约能优化 XX 万成本。 不知道贵公司对稳定性和成本这块关注度大吗？方便的话我们约个 20 分钟的视频沟通？"
Day 7 · 面谈/视频会议 展示"无感迁移"方案，解除顾虑 关键节点
目标：消除客户最大顾虑（迁移麻烦、数据风险），建立深度信任
准备 5 页演示材料：问题确认 → 方案对应 → ROI 测算 → 迁移说明 → 下一步
展示迁移代码对比图：只改 base_url 一行，其他代码 100% 不动
承诺 POC 测试：7 天免费专有集群体验，先跑 1% 流量
邀请技术负责人参与（帮销售锁定真实决策人）
面谈结束时的话术
"我今天带了一个 7 天免费测试账号，您的技术同学今天就能接入，只需要改一行代码—— base_url 换成我们的地址，其他什么都不用动。先切 1% 的流量过来，跑 3 天， 我给您一份详细的对比报告。没有任何风险，您看要不要现在就开始？"
Day 10 · POC 测试中 主动提供数据报告，建立信任感 数据说话
目标：用 POC 实测数据替代销售说辞，数字比话术更有力量
主动生成对比报告：平均响应时间、成功率、成本估算
每 2 天主动更新数据，显示专业态度
如有任何技术问题，4 小时内必须响应解决
POC 阶段数据汇报话术
"王总，POC 跑了 48 小时，给您汇报一下数据：平均响应时间从 2.4 秒降到 0.8 秒， 成功率 99.97%，按这个表现折算月成本大约能省 XX%。 数据都有日志，随时可以调阅。您这边技术团队有什么反馈吗？"
Day 15 · 逼单 限时额度/阶梯价锁定，推动签约 成单冲刺
目标：在客户信任高点、POC 数据已验证时，用限时优惠促成签约
提出限时优惠：本季度签约享受阶梯优先价 + 额外 Tokens 赠送
提前准备好合同草案，减少客户"等一等"的理由
如有必要，申请邀请上级一起拜访（增加重视感）
逼单话术（在 POC 数据超预期后使用）
"王总，POC 结果很好，您技术团队也认可了。我这边整理了一个方案， 本季度签约可以锁定优先价，还额外赠送 XX 万 Tokens 的测试额度， 相当于帮您多省了一笔。我们合同很标准，一般 2 天就能走完。 您看本周能确定下来吗？我今天就把合同草案发您。"

谈判技巧 · 异议处理

客户每一个"拒绝"背后都藏着一个"需求"，掌握这12种翻转话术，将异议变成成交契机

🔍
先认同，再转折
永远先表示理解客户的顾虑，再引出你的解决角度。"您说得有道理，不过……"比直接反驳效果高 3 倍。
情感账户优先
💡
用数字说话
任何主张都要有数据支撑。"更稳"不如"高峰期 99.9% 可用率"，"更省"不如"月均降本 30%"。
量化优势
🧪
POC 化解一切
遇到任何疑虑，第一反应是"我们先测试验证"，测试数据比销售语言更有说服力，客户自己看到就信了。
数据胜于言辞
🎯
聚焦价值不聚焦价格
客户说贵时，立即把对话拉回价值：节省的钱 + 稳定带来的收益。让"贵"的感觉消失在 ROI 里。
价值优先
⏳
降低决策风险
客户犹豫 = 风险感太高。你的任务是把风险降到最低："先迁 1%，不影响现有业务，随时可以回滚"。
风险管理
🤝
服务感 > 推销感
始终以"我来帮您解决问题"的顾问心态，而非"我要卖给你东西"的推销心态。客户感受到服务温度，信任自然建立。
顾问式销售
高频异议翻转卡
"官网直接用就挺好的，为什么要换？"
标准应对思路
官网采用共享集群，高峰期所有用户争抢同一资源。我们提供专有集群，独占通道， 高峰不降速、不排队。等到客户因为宕机损失了一笔收入，再换就晚了。
"官网确实方便，很多公司早期都这样用。但是您有没有注意到，每次高峰期——比如促销活动、内容爆发期——AI响应速度会有明显下降？那是因为官网是共享集群，大家一起用。我们提供专有集群，就像给您开了一条专用车道，别人堵车不影响您。我们先跑个测试对比，您自己看数据更直观。"
"价格太贵了，官网价格比你们低。"
标准应对思路
不直接降价，把对话转移到总体成本：我们的智能路由 + 智能记忆层可以综合降本 30%， 再加上稳定性带来的业务收益，实际 TCO（总拥有成本）远低于官网。
"您说的单价，我们确实不是最低的。但我想帮您算一笔账：我们的智能路由会自动选最优性价比的模型，智能记忆层对相似请求复用结果——这两项加起来，实际调用成本平均降30%。按您每月XX万Tokens的量，一年节省的金额比价差大多了。我们先测试7天，按实际数据来谈，更有说服力。"
"迁移太麻烦了，我们不想改代码。"
标准应对思路
迁移只需要改一行代码：把 base_url 换成我们的地址，其他接口参数、请求格式完全一致， 技术团队 10 分钟内完成，不影响任何现有业务。
"我理解您的顾虑——改代码确实是个风险。但实际上，我们的接入方式和官网 API 完全兼容，唯一要改的就是一行代码：把请求地址换成我们的。字面上就是改一个 URL，其他所有参数、返回格式都不变。我们还提供接入说明文档，您的技术同学10分钟就能搞定。您看要不要让技术负责人加一下我，我直接给他发文档？"
"我们自己有技术团队，能直接对接官网。"
标准应对思路
自建技术能力是优势，但多模型管理、智能路由、故障切换这些工作需要大量工程投入。 我们帮客户省掉这个成本，让技术团队聚焦核心业务，而非维护基础设施。
"您的技术团队很强，这是好事。问题是，维护多模型接入、做智能路由、处理故障切换——这些工作本身就是一个独立的工程项目，需要持续投入人力。我们把这些都做好了，您的技术团队可以把精力放在产品本身。相当于您节省了一个工程师的成本，同时还得到了更好的稳定性和更低的调用成本。"
"我们现在没有预算，下个季度再说。"
标准应对思路
先跑 POC 不需要预算，等下季度预算到位时，已有 3 个月的数据支撑， 内部审批更容易，还能锁定当前的价格。
"完全理解，预算这块要等审批。不过有个建议：POC 测试是免费的，不占预算。先跑起来，等您下季度申请预算时，手里有3个月的实测数据，比光凭我说好用要有说服力得多，审批通过率也高很多。您觉得先把测试跑起来怎么样？"
"我们的数据很敏感，不敢放到第三方平台。"
标准应对思路
我们是阿里云核心代理商，底层仍然是阿里云基础设施，同等安全级别。 业务数据只是 Tokens 请求的路由转发，不存储客户的业务内容。

高阶谈判动作 · 推进决策的5个技巧
假设性提问
"如果测试数据达到您的预期，您这边决策大概需要多长时间走完流程？"——帮助销售提前了解决策链，同时激活客户的"假设成真"心理。
紧迫感制造（真实的）
"本季度末专有集群价格会随成本调整，现在签约可以锁定当前价。"只用真实的优惠或资源稀缺性，不夸大，不造假。
第三方背书
"同行的某AI公司上个月刚迁过来，CTO亲自说延迟降了65%。"同类企业的真实案例比任何广告都有效，优先使用同行业案例。
微承诺积累
从"加个微信" → "看一份资料" → "约个视频" → "跑个测试"，每一步都是小承诺，小承诺积累到 POC，签单就是顺水推舟。
沉默是金
报价或提关键问题后，保持沉默，等客户先开口。很多销售因为紧张而自己打破沉默，反而削弱了谈判筹码。沉默往往会带来更好的条件。
绝对禁止
不说"我们比阿里云便宜" → 违反合规，且给客户错误定位
不说"官网很卡" → 贬低上游，伤害代理商形象
不承诺"100%不掉线" → 无法兑现，伤害信任

闭坑指南

先人踩过的坑，你不必再踩一遍。以下是造成丢单最多的高危场景与行为，牢记于心

话语雷区 · 禁止说 vs 必须这样说
"我们比阿里云便宜！"
"我们帮您优化成本结构，综合下来每年能省 XX 万。"
⚠️ 危险原因：代理商不能公开与上游竞价，且会让客户认为你只是靠便宜吃饭，一旦有更便宜的立即被替换。
"官网很卡，经常出问题！"
"共享集群在高峰期资源是有限的，我们的专有集群能给您独立保障。"
⚠️ 危险原因：贬低阿里云官网会反噬代理商信誉，客户会质疑你的专业性和合规性。
"保证 100% 不掉线，永不故障！"
"多重架构保障，SLA 承诺 99.9%+ 可用率，故障时自动切换备用节点。"
⚠️ 危险原因：无法兑现的承诺一旦出问题，客户投诉、退款、口碑破产，一单变成负资产。
"我们是市场上最便宜的！"
"我们的定价在市场上有竞争力，加上专有集群和智能路由，综合性价比很高。"
⚠️ 危险原因："最便宜"定位会吸引价格敏感型客户，忠诚度极低，且拉低品牌价值。
"这个我不太清楚，我回去问问。"（每次都这样）
"这个技术细节我稍后让技术同事直接和您确认，今天先把核心方案敲定。"
⚠️ 危险原因：频繁说"不知道"会让客户失去信心。先稳住节奏，及时跟进，而不是当场暴露无知。
"您考虑一下，想好了联系我。"（然后消失）
"您考虑方面，我周三发您一份数据报告，您看完有问题随时找我。"（锁定下次接触）
⚠️ 危险原因："等客户联系你"是销售自杀行为，优质客户的注意力极其短暂，3天没跟进就凉了。
高危丢单场景 · 识别 + 应对
🎯
场景一：客户根本没在用 Tokens
打了30分钟电话，对方说"AI我们还在研究阶段，现在没有调用"。这不是目标客户， 是浪费时间的大坑。不要试图"培育"他，当前阶段没有购买决策能力。
正确处理：礼貌结束，加入"培育名单"，3个月后再回访。立即转移精力到真正在用 Tokens 的客户。
💰
场景二：一上来就谈价格
客户第一句话就问"你们多少钱一百万 Tokens？"。如果你直接报价，会陷入纯价格竞争， 你将永远赢不了比你更便宜的竞争对手。
正确处理：先说"价格根据用量和配置不同，让我先了解一下您的场景，我给您出一个最优方案，报价才有参考意义"，把对话引向需求挖掘。
🧱
场景三：只联系到执行层，找不到决策人
和技术工程师聊了很久，对方很感兴趣，但采购决策权在 CTO 或技术总监， 工程师无法推动，客户陷入"研究中"状态，永远没有下文。
正确处理：让工程师帮你"向上传递"——"您看能不能帮我约一下 CTO/技术总监，我来给他直接演示方案，这样您内部推进也更容易"。让内部人帮你搭桥。
😴
场景四：方案讲得太复杂，客户听不懂
花了20分钟讲架构、TPM、专有集群、智能路由……客户沉默，其实是听不懂。 技术名词多了，客户只会觉得麻烦、风险高，更不敢迁移。
正确处理：只讲三个数字：快多少%、省多少钱、迁移多简单。其他技术细节留给客户技术团队对接时再说，销售阶段永远只打"业务价值"。
⚡
场景五：POC 跑得不好，数据没达预期
测试期间出现延迟偏高、偶发错误，客户对结果不满意，信任急速下降， 丢单概率极高。这是最致命的场景，一旦发生必须立即响应。
正确处理：第一时间联系技术支持，4小时内给出定位原因+优化方案；主动向客户说明+道歉；提出延长测试期补偿。绝不沉默，主动出击挽回。
