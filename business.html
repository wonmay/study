问对问题，1分钟内定位客户档位，报出对应的价值方案

怎么问出客户的月 Token 消耗量？

直接问"月消耗量"客户往往不知道怎么回答。换个问法效果更好：

「您们现在每个月在大模型 API 上大概花多少钱？几百、几千、还是更多？」
→ 几百元：轻量起步档（< 100万 Token）
→ 几千到几万：企业 API 档（100万～1000万）
→ 几万到几十万：定制方案档（1000万+）
客户说"我不知道我用多少 Token"，怎么办？

换一个角度问，帮客户估算：

「您们的 AI 功能，一天大概有多少用户在用？每次对话大概几个来回？」
→ 日活 500 人 × 10 轮对话 × 每轮 500 Token = 每天 250万 Token
→ 月均约 7500万 Token，属于企业 API 档高段位或定制档

💡 用 ROI 计算器 帮客户现场估算，比口说更有说服力。

客户用量很小（< 10万/月），值得跟进吗？

单纯看当前用量不够，要看业务增长趋势：

  • ✅ 产品刚上线 AI 功能，用量必然快速增长 → 值得提前签，锁定长期合作
  • ✅ 正在做 POC 测试，准备大规模上线 → 当前只是前期，签约时机正好
  • ❌ 只是个人开发者或临时项目 → 不是目标客户，直接放弃

三大痛点 · 对应解决方案

客户说出下面这些话，立刻对应到方案——不要犹豫，直接给答案

😤 痛点 1：「高峰期 AI 响应很慢，有时候直接超时，用户反馈差」

根因：官网共享集群高峰期资源不足，TPM 被限流。

rhcloud 方案：专有集群独享资源，TPM 有保障，高峰期不影响。

「这是共享集群的通病，高峰期大家抢资源。我们给您的是专属通道，您自己用自己的，不管其他用户多少，您的响应速度和稳定性不受影响。我们可以先做测试，让您自己看数据对比。」
💸 痛点 2：「AI 成本占我们营收比例越来越高，老板在问能不能降」

根因：单一模型、没有路由优化，所有请求用最贵的模型处理，造成浪费。

rhcloud 方案：智能路由 + 智能记忆层，简单请求走便宜模型，复杂请求才用高级模型，重复问题直接复用缓存。

「您现在是不是所有请求都走同一个模型？其实简单的问答根本不需要用最贵的模型，我们的系统会自动判断——简单问题给低价模型，复杂任务才用高级模型。平均下来综合成本能降 20~35%，我给您算一下具体数字。」
🔧 痛点 3：「我们用了好几个模型，账号、账单、接口各管各的，很乱」

根因：多厂商模型各有独立API，接口格式不统一，管理成本高，数据分散无法汇总。

rhcloud 方案：统一入口接入，全部模型一个API Key 调用，用量和账单统一管理，技术团队不再维护多套接口代码。

「您现在维护几套 API 接口？每个模型单独的 Key、单独的账单、接口格式还不一样。我们帮您统一到一个接口，所有模型都能调，账单统一看，您的技术团队代码量减少一大半。」
