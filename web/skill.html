01

系统运行原理

OpenClaw 是一套部署在您个人服务器上的 AI 期权数据参谋系统，基于开源 OpenClaw 框架搭建。 它将多个专业数据源的信号统一汇聚，经过量化引擎精确计算后，在满足预设策略条件时通过飞书或 Telegram 主动推送给您。 所有交易决策最终由您人工判断，系统绝不自动下单。

Layer 1 · 数据层
多源数据采集（9层冗余）
系统通过脚本同时接入多个数据渠道：机构期权流平台（OptionWhales、Unusual Whales）、GEX 分析平台（Profitabul）、 期权链数据（Alpaca、Massive、Finnhub）以及股票行情（Alpha Vantage 多 Key 轮询 + yfinance 备用 + Yahoo 抓取兜底）。 任一源故障，系统自动切换，不中断服务。
↓
Layer 2 · 计算层
Python 量化引擎精确计算
所有 Greeks（Delta、Gamma、Theta、Vega）、IV Rank、GEX 分析、年化收益率，均由服务器上的 Python 脚本 通过 Black-Scholes 模型精确计算，AI 不允许口算或估算，保证数据严谨性。
↓
Layer 3 · 决策层
策略合规检查 + 人工确认门
AI 参照您预先设定的策略文档（开仓条件、风控规则、仓位比例），逐条核验信号是否达标。 推送前必须等待您回复「确认」，系统才会发出完整策略细节，不会单方面推进。
↓
Layer 4 · 通知层
飞书主推 / Telegram 备用
符合条件的商机推送格式化消息到您的飞书或 Telegram，包含：标的、行权价、成交价、到期日、年化收益率、完整 Greeks 及开仓理由。 无商机时绝对静默，同一机会 24 小时内不重复推送。
↓
Layer 5 · 记忆层
持仓历史 · 复盘分析 · 策略进化
系统通过 SQLite 数据库记录每次调仓与持仓变化，支持对比历史调仓差异。 AI 每日生成复盘摘要，发现的规律以建议形式提交，由您决定是否修改策略文件，AI 无权直接更改规则。

Delta、Gamma、Theta、Vega 全部由服务器 Python 脚本通过 Black-Scholes 模型精确计算。 每一条推送均附完整 Greeks，让您在开仓前清楚知道风险敞口，不再靠直觉拍脑袋。

系统内置交易验证器：在推送任何信号前，自动逐条检查仓位比例、财报日禁令、VIX 风控门槛、年化收益是否达标（≥30%）。 您收到的每一条推送，都已经通过了策略层面的预筛选。

持仓记忆与历史对比
每次调仓记录存入 SQLite，可随时查询「这次调仓和上次差异在哪」。AI 每日自动复盘，将发现的规律以建议形式提交， 策略优化权始终在您手中，AI 不可直接修改规则。

主动推送，无需盯盘
您只需在新加坡时间晚上留意飞书通知——有商机即推送，无商机绝对静默。 系统全程运行在服务器，与您的设备无关，24/7 不间断监控。
🛡️
人工确认门，风控铁律
任何交易建议推送前，系统必须先收到您回复的「确认」指令才会继续。 禁止裸卖空、单仓≤5%、财报前3天禁止开仓等铁律写入系统灵魂文件，AI 无权逾越。

解决期权投资者的六大痛点

😵
痛点一：数据太分散，每天要看十几个平台
→ 系统自动汇聚机构期权流、GEX、期权链、股价行情，每天开盘后扫描一次，结论直接推送到飞书，您只需看一条消息。
🤯
痛点二：Greeks 计算复杂，容易算错或靠估
→ 所有 Greeks 由 Python 脚本精确计算并附在推送里，您在决策时有完整的数学依据，不再靠感觉。
📅
痛点三：财报日、VIX 异动等风险节点容易遗漏
→ 系统自动查询财报日，财报前 3 天内禁止开仓信号推送；VIX > 30 时自动将仓位建议降低 50%。
🎯
痛点四：年化收益不佳的机会占用注意力
→ 系统只在年化收益 ≥ 30% 时推送，其余信号静默过滤，精准捕捉高收益窗口。
📊
痛点五：想验证交易计划是否符合自己的策略规则
→ 发送「验证交易计划」，系统会三层逐条核查：市场环境 → 策略合规 → Greeks 计算，输出明确的执行/等待/放弃建议。
📚
痛点六：复盘和调仓对比耗时费力
→ 每次调仓自动存档，可随时查询历史对比；AI 每日生成复盘摘要，发现规律后以建议形式提交供您审阅。

使用前提条件

1
期权交易基础知识
您需要理解期权的基本概念：Call / Put、行权价、到期日、权利金， 以及 Greeks（Delta 方向敞口、Theta 时间衰减、Vega 波动率敏感性、Gamma 加速度）的含义。 系统提供精确数据，但解读和最终决策需要您具备相应的专业判断力。 本系统面向有期权交易经验的投资者，不适合完全零基础的初学者。
2
一台可 24/7 运行的服务器
系统需要部署在持续运行的服务器上，推荐使用云端 VPS（DigitalOcean、AWS、Vultr 等）， 操作系统为 Linux（Ubuntu 22.04）。不建议部署在个人电脑，因为定时任务和推送服务需要不间断运行。
3
Anthropic API Key（Claude）
系统的 AI 推理核心使用 Claude Sonnet API。您需要在 console.anthropic.com 注册账号， 充值并获取 API Key，月均消耗约 $20–30。这是系统运行的必要前提，无法绕过。
4
飞书或 Telegram 账号（推送渠道）
推送通知通过飞书（主）或 Telegram（备）发送。您需要提前创建飞书机器人 Webhook 或 Telegram Bot， 并将对应的 Webhook 地址配置到系统环境变量中。飞书在新加坡使用体验更好，推荐作为主推送渠道。
5
基础 Linux 命令行操作能力
系统部署、脚本运行、环境变量配置等操作需要在命令行（Terminal）中完成，通过脚本或代码实现自动化流程。 您不需要是专业程序员，但需要能够按照部署指南执行基础命令、编辑配置文件和查看日志输出。
6
至少一个数据源 API Key（期权数据）
系统支持 9 层数据源，但至少需要一个可用的期权数据 API（Massive API、Finnhub 或 Alpaca 均可）。 股票行情数据可通过免费的 yfinance 和 Yahoo Finance 抓取兜底。 验证阶段可先用免费资源，确认系统稳定后再逐步升级付费数据源。
7
明确的个人期权投资策略
系统的策略合规检查以您预先定义的策略文档为准。您需要在部署前明确： 关注哪些标的、风险偏好（最大单笔亏损比例）、期望的年化收益门槛、持仓周期、 以及允许使用哪些期权策略类型（如 Bull Call Spread、Covered Call 等）。 系统可参考 bizsn.com/free/options 策略库自动学习并生成对应扫描脚本。

系统铁律与安全边界

以下规则写入系统灵魂文件（SOUL.md），AI 无权修改，人工定期审查。 这些约束是系统设计的核心价值观，确保工具始终处于可控状态。

✅ 只做分析，不下单
AI Agent 永远不触碰实际交易执行接口，与您的券商账号完全隔离。系统不连接 Interactive Brokers 或任何券商的下单 API。
✅ 多源交叉验证
所有交易信号需多个数据源确认方才推送，单一数据源的异常信号不会触发推送，降低误报风险。
✅ 人工确认门
系统在推送任何完整策略建议前，必须收到您明确的「确认」回复。AI 无法在没有人类确认的情况下推进下一步。
🚫 禁止裸卖空期权
Naked Put 和 Naked Call 在策略层面被明确禁止，系统不会生成此类交易建议，无论市场条件如何。
⚡ 单仓 ≤ 账户 5%
任何单一期权策略的仓位不得超过总账户的 5%，单笔最大亏损不超过账户 1%。系统在验证器层面强制执行，违反者不推送。
⚡ 财报前3天禁入
财报日前 3 天内，系统自动禁止生成新开仓建议。财报窗口的 IV 变化和价格跳空风险不在系统的风控模型范围内。
⚡ VIX > 30 自动降仓
当市场恐慌指数超过 30 时，系统自动将仓位建议降低 50%，帮助您在高波动市场中保持更保守的仓位管理。
✅ Greeks 禁止口算
所有 Greeks 必须通过 Python Black-Scholes 引擎精确计算，AI 不被允许凭语言模型估算，确保数字严谨性。
✅ AI 无法修改策略
系统发现的交易规律以建议形式提交给您，核心策略文件（TRADING_RULES.md）的修改权完全在您手中，AI 只能读取不能写入。
关于系统的长期演进： 随着您使用时间的增长，系统会通过每日复盘不断积累市场洞察，形成「经验闭环」。 AI 将规律性发现写入待讨论列表 → 您决定是否纳入策略 → 人工修改规则文件 → 通过 Git 版本控制追踪变更 → 新参数回测验证后方才生效。 这套流程确保系统在持续学习的同时，始终保持人类的最终控制权。
