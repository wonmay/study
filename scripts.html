二、痛点挖掘话术 7条 · 引导客户说出稳定/成本/速度痛点
PAIN-01 · 稳定性路径
从"高峰期响应"切入稳定性痛点
稳定性 电话/面谈
适用：客户有高并发业务场景（直播、视频、教育）
"我想先了解一下您们现在的使用情况——高峰期的时候，大模型响应速度大概是多少秒？有没有出现过超时或者报错？这对业务影响大不大？"
"高峰期"是触发痛点的关键词；让客户描述具体场景，痛点自然浮现。
PAIN-02 · 成本路径
从成本占比切入，引导成本意识
成本 电话
适用：已知客户月消耗量较大，成本可能是关注点
"现在大模型 API 的费用大概占贵公司整体运营成本多少比例？随着业务量增长，这块的成本压力大不大？有没有想过优化一下？"
问"比例"而非"金额"更容易让对方开口；"增长"暗示未来压力会更大。
PAIN-03 · 多模型路径
挖掘多模型管理和切换的痛点
多模型 面谈
适用：技术团队较成熟，已在使用多个AI厂商
"您们现在大模型 API 这块是只用一家的服务，还是有在用多家？如果要切换模型或者某个模型出问题，切换起来方不方便？"
用"方便不方便"引出切换麻烦，自然铺垫统一接口的价值。
PAIN-04 · 开放式挖掘
直接开放式提问，让客户说出第一痛
开放式 面谈
适用：已建立一定信任，客户比较开放时使用
"如果我问您一个问题：在大模型这块，您现在最头疼的是什么？是稳定性、成本还是速度，或者别的方面？"
给出三个选项降低客户思考成本；"或者别的"留出余地，避免框死方向。
PAIN-05 · 业务影响放大
从技术问题引入业务损失，放大痛点
影响放大 面谈
适用：客户承认有过稳定性问题，进一步放大
"您们做 AI 这块，有没有碰到过——用户在某个时间段，AI 回复特别慢甚至没有响应的情况？这种时候，用户的流失率有没有影响到业务指标？"
SPIN 中的 Implication（影响）阶段；把技术问题和业务 KPI 联结，让痛点变"贵"。
PAIN-06 · 未来锁定风险
引导思考"未来切换的痛"，制造前瞻焦虑
未来风险 面谈
适用：客户当前满意现有服务，需要创造未来需求
"您们现在用的大模型，是固定用某一个，还是会根据不同场景选不同的模型？如果以后出了更适合你们业务的新模型，切换的成本大吗？"
让客户想象未来的麻烦；聪明的 CTO 会认识到锁定单一厂商的风险。
PAIN-07 · 需求引出
SPIN 的 Need-payoff：让客户描述解决后的价值
SPIN法 面谈
适用：痛点已充分挖掘，准备过渡到方案介绍之前
"如果现在大模型这块稳定性、成本、速度这三个问题都能解决，您觉得对公司的业务帮助大不大？换句话说，这三块如果都优化好了，能帮你们拿到多少业务增长？"
让客户自己说出解决方案的价值，比你讲有说服力10倍；为后续报价锚定预期。
三、面谈沟通话术 6条 · 用于视频会议或线下拜访场景
MEET-01 · 开场确认
面谈开场：复述客户情况，建立专业感
面谈 专业感
适用：视频会议或面谈开场，复述已了解的信息
"今天咱们聊一下，我先整理一下我对贵公司情况的理解，有不准确的您帮我纠正。您们目前 AI 业务月均调用量大概在 XX 级别，主要用在 XX 场景，主要关注稳定性和成本这两块，对吗？"
面谈必须展示"功课做足"；客户会感受到被重视，也会帮你纠正信息偏差。
MEET-02 · 方案呈现
方案呈现：痛点对应，数字说话
面谈 方案提案
适用：痛点已确认，进入方案介绍阶段
"根据您刚才说的情况，我给您梳理一下。您现在的问题主要是两个：一是高峰期响应不稳，二是成本有点高。我们的方案：第一，接入专有集群，不与其他用户共享，高峰期独立通道；第二，智能路由自动选最优性价比模型，综合成本降 30%。迁移只需改一行代码，当天可以完成，零风险。"
先"问题" 再"方案"，顺序不能反；每个方案点对应一个痛点，对位要精准。
MEET-03 · ROI 现场计算
当场帮客户算 ROI，让数字自己说话
ROI计算 数字驱动
适用：已获得客户月用量和单价信息后使用
"我帮您算一下。您每月消耗 XX 万 Tokens，现在单价是 X 元/百万，一年花了 XX 万。切过来之后，综合成本能降 30%，一年省 XX 万。加上稳定性提升减少的用户流失，总体收益还会更高。您觉得这个账算得过来吗？"
当场手算比PPT更有说服力；最后反问让客户自己肯定价值，成交更顺。
MEET-04 · 迁移说明
用最简单的话解释迁移有多容易
技术迁移 消除顾虑
适用：客户对迁移工作量有顾虑时使用
"迁移这件事其实比您想象的简单很多。您的代码里有一个 base_url，把它从阿里云官方地址换成我们的地址，其他参数、请求格式、返回格式完全一样，不需要改任何业务代码。您的技术同学今天就能操作完，30 分钟搞定。"
"30分钟"是具体数字，能有效打消"很麻烦"的直觉；主动提供文档展示服务态度。
MEET-05 · POC 邀请
POC 测试邀请：零风险，数据驱动决策
POC测试 降低风险
适用：客户有兴趣但尚未下决心，准备推进 POC
"我给您申请一个 7天的专有集群测试账号，先切 1% 的流量过来跑。不影响现有业务，随时可以回滚。跑 3 天我给您出一份数据对比报告，您看完数据再决定要不要全量迁移，完全没有压力。"
"1%"和"随时回滚"把风险降到最低；数据报告让 POC 有明确输出，推进成功率极高。
MEET-06 · 生态价值
介绍多模型聚合的长远价值
多模型 面谈
适用：技术决策人关注长远架构灵活性的场景
"我们还有一个多模型聚合的能力——您现在用的这个模型，如果以后出了更好的、更便宜的新模型，通过我们的平台可以直接切换，不需要改代码。这相当于给您的技术团队省去了大量的对接工作，以后随时用市场上最好的模型。"
CTO 关心灵活性和技术负债；"不需要改代码"是最直接的价值表达。

真实稀缺感+了解流程，推进决策
紧迫感 关单
适用：客户有意向但在拖延，需要制造紧迫感
"我们本月还有最后 X 个专有集群的优先席位，申请满了之后下个月新客户要等资源。您们的用量确实适合专有集群方案，我帮您提前锁一个。您这边签约流程大概要走多久？"
只用真实的稀缺性，不夸大；问"流程走多久"是在推进而非催单，体验更好。
CLOSE-03 · 顾虑清零
逐一确认顾虑已解决，清除成交障碍
顾虑消除 异议处理
适用：客户已了解方案，但还在犹豫，帮助做最后决策
"我帮您理一下：您现在的顾虑是不是就两个——一是价格，二是迁移风险？价格这块我刚才算了，综合下来比现在省；迁移风险我们有 POC 保证，1% 流量先跑，不满意随时回滚。这两个问题都解决了，您还有什么需要确认的吗？"
"还有什么需要确认的"是假设性关单法——默认一旦无顾虑就要推进，把球踢给客户。
CLOSE-04 · 小单切入
以小额试用单切入，降低决策门槛
分步签约 关单
适用：客户有意向但内部审批慢，用小单切入建立合同关系
"我们可以先签一个小额试用合同——比如先买 3 个月、100 万 Tokens 的套餐，验证完效果再升级。这样您们内部审批也快，风险可控，又能早点把成本优化跑起来。您觉得这个方案可行吗？"
建立合同关系是关键里程碑；小单进大单是经典路径，客户心理摩擦最小。
高转化率
CLOSE-05 · 时间线关单
给出清晰时间线，帮客户"看见成果"
时间线 关单
适用：客户情绪和信任均到位，做最后一步推动
"您今天能拍板吗？我们这边可以给您出正式报价单和合同，您今天签意向，下周就可以交付专有集群账号和接入文档，您的技术团队下周就能开始迁移测试。我帮您把时间节点写清楚。"
直接问"能拍板吗"，坦诚不做作；时间线让客户想象"下周就开始省钱"，动力更强。
六、跟单跟进话术 3条 · 保持活跃度、重新唤醒沉默客户
FOLLOW-01 · Day3 跟进
Day 3 · 带新价值（数据+动态）回访
Day 3 微信
适用：首次接触 3 天后的跟进，带新信息回访
"王总好，上次和您聊完已经过了三天了。我整理了一份同行业客户的实测数据报告，您看一下有没有参考价值？另外最近大模型有个新版本上线，我们平台已经接入，性价比更高，您有需要的话我给您发接入说明。"
每次跟进必须带新价值，不做无意义催单；"新版本"是天然的联系借口。
FOLLOW-02 · 沉默唤醒
一周未回应 · 礼貌唤醒+主动帮解释
唤醒 微信/电话
适用：客户超过1周无回应，礼貌唤醒
"王总，打扰一下，想再问一下上次聊的大模型优化方案，您那边有没有进展？如果内部讨论有什么问题，我可以来配合解答。或者您看方便的话，我和您们技术负责人直接沟通一下，帮您们省点内部解释的时间？"
主动帮客户做内部工作，是服务感的体现；提出和技术负责人对接，帮助绕开沟通瓶颈。
FOLLOW-03 · POC 汇报
POC 结束 · 数据汇报并推进签约
POC汇报 数据驱动
适用：POC 测试结束，发送数据汇报并推进下一步
"王总，我们 POC 数据跑完了，给您发一份对比报告。核心数据：平均响应时间从 2.4s 降到 0.9s，成功率 99.97%，按您们的月用量折算，切换后每月节省约 XX 元。您看完有问题随时找我，如果效果满意，我们可以聊下一步的正式合作方案了。"
数字越具体越有说服力；结尾"可以聊下一步"是自然过渡到签约的衔接语。
