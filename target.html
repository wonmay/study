拜访前 10 分钟侦查清单

每次拜访前过一遍，带着答案去谈，效率提升 3 倍

步骤 1 · 企查查 / 天眼查查经营范围（2 分钟）

搜索公司名，查看「经营范围」是否包含：

人工智能 大模型 智能科技 AI 应用 自然语言处理 算法

顺带看：注册资本、成立时间、融资情况——这些帮你判断决策效率和付费意愿。

步骤 2 · Boss直聘搜招聘信息（3 分钟）

在 Boss直聘 搜公司名，筛选在招岗位，重点看：

  • ▸ 大模型工程师 / AI 算法工程师：直接用量信号
  • ▸ 岗位描述里提到 API 对接 / RAG 开发 / Prompt 工程：必然在消耗 Token
  • ▸ 招聘数量越多，代表 AI 业务规模越大
步骤 3 · 打开官网找 AI 功能入口（2 分钟）

官网产品页 / 功能介绍页，找这些关键词：

AI 客服 智能助手 AI 写作 大模型驱动 智能推荐 AI 生成

如果找到了，拜访时直接问：「我看到你们产品里有 XX 功能，这块的 AI 调用是你们自己接的 API 还是用的第三方服务？」

步骤 4 · 确认联系人层级（3 分钟）

通过 LinkedIn、脉脉、官网团队页确认目标联系人职位。目标优先级：

🥇
CTO / VP Engineering / 技术总监 ——最佳决策人
🥈
AI 负责人 / 架构师 / 算法组长 ——可影响决策
🥉
采购 / 运营 / 行政 ——需要绕行，争取转介到技术层

把这张画像贴在桌上——每次打电话前看一眼，想想对面的人是不是这种人

六大排除类型 · 别浪费时间

个人开发者
用量极小（月均 < 1万 Token），价格极度敏感，决策极慢。投入产出比为负，放弃。
🏛️
政府 / 国企单位
采购必须走招标流程，从立项到签约通常 6~18 个月，销售资源严重错配，不适合现阶段。
🏢
头部互联网大厂
百度 / 字节 / 阿里 / 腾讯等，有自建大模型基础设施，不需要外部代理服务，且议价空间极低。
🔬
大模型研发原厂
DeepSeek / 百川 / Moonshot 等，他们就是我们的供应商或竞争对手，无法成为客户。
📱
只用 C 端 AI 工具
用钉钉 AI / 飞书 AI / WPS AI 的企业，是工具用户不是 API 用户，不消耗 Tokens，不在目标范围。
❄️
无 AI 业务规划的传统企业
没有 AI 产品、没有 API 用量、没有技术团队——需求还没产生，教育成本极高，放弃。
工具使用说明
电话 / 拜访结束后，花 30 秒回答 5 道题，系统即时给出 A/B/C 类判断 + 配套下一步动作 + 话术。共 15 分满分。
第 1 / 5 题
第 1 题 · 招聘信号
这家公司最近有招聘 AI 相关岗位吗？
在 Boss直聘 / 拉勾 搜索公司名后判断
有在招「大模型工程师」「AI 算法工程师」，且有多个岗位 +3 分
有 AI 相关岗位，但数量少或描述比较泛（"了解 AI"） +2 分
有招技术岗，但没有明确 AI / 大模型方向 +1 分
没有在招技术岗，或根本找不到招聘信息 +0 分
第 2 题 · 技术栈
他们的技术团队有没有对接过 AI API？
可通过官网职位描述、GitHub、或直接询问判断
明确有 API 对接经验，提到 OpenAI / 通义 / DeepSeek 接入 +3 分
有 AI 功能，但不确定是 API 自研还是第三方 SaaS 封装 +2 分
官网提到 AI，但技术层不清楚，需要进一步确认 +1 分
没有任何 AI 技术栈信息，或明确表示用的是第三方 AI 工具 +0 分
第 3 题 · 官网 AI 功能
公司官网 / 产品页有没有 AI 功能入口？
打开官网，看产品功能列表是否有 AI 相关描述
有完整 AI 功能板块，且可以直接体验（有 Demo 或登录入口） +3 分
官网提到了 AI / 智能功能，但只有文字描述没有入口 +2 分
官网经营范围含 AI，但产品里找不到具体的 AI 功能 +1 分
官网没有任何 AI 相关描述，产品也看不出 AI 应用 +0 分
第 4 题 · 月用量估算
沟通后，你对他们的月 Token 用量判断是？
可直接问月均 AI 花费，或通过用户量 × 功能使用频率估算
月用量 100万～5000万 Tokens（月费 2000～10万元），且持续增长 +3 分
月用量 10万～100万 Tokens（月费 200～2000元），AI 业务在发展 +2 分
月用量 1万～10万 Tokens，目前只是测试阶段，还未正式商业化 +1 分
用量极低 / 不确定是否有用量 / 无法获取相关信息 +0 分
第 5 题 · 决策人层级
你接触到的联系人，对 AI 采购的决策权是？
判断此人能否最终拍板签单，或需要向谁汇报
是 CTO / VP Engineering / 技术总监，可以直接拍板采购 +3 分
是 AI 负责人 / 架构师，有强烈话语权，需向 CTO 汇报 +2 分
是技术工程师，有意愿推动但需要多级审批才能签单 +1 分
是采购 / 运营 / 行政，不了解技术需求，决策链极长 +0 分
